How to Use the Simulator

Purpose of the Application

This browser-based application is a fully functional Artificial Neural Network (ANN) simulator. It allows you to design a custom network architecture, train it on your own datasets, visualize the learning process in real-time, and utilize the trained model to make predictions on unseen data.

Workflow & Constraints

  • 1. Build Network: Define the number of input nodes, output nodes, hidden layers, and nodes per hidden layer. Select your desired activation functions. Click "Build Network" to instantiate the model and visualize the uninitialized architecture.
  • 2. Train Network: Upload a properly formatted Training CSV. You can toggle "Normalize Inputs" to mathematically scale your input features between 0 and 1, preventing features with large numerical ranges from skewing the weights. Adjust your Learning Rate and Epochs, then click "Train Network" to begin gradient descent.
  • 3. Download Weights: Once trained, you can export the exact weight and bias matrices as a CSV file for your records or external analysis.
  • 4. Run Predictions: Upload a Testing CSV containing new, unseen data. If you normalized your training data, the application will automatically scale this testing data using the exact same min/max parameters discovered during training. Click "Run & Download Predictions" to receive a CSV mapping your inputs to the network's final output.

Formatting Your CSV Data

To ensure the application reads your data correctly, files must strictly adhere to the following rules:

  • Headers: You may include an optional top row for column headers (e.g., "Age, Height, Income"). The application will automatically ignore the first row if it detects non-numeric strings.
  • Training Data: Your training CSV must contain exactly Input Nodes + Output Nodes columns. The input columns must come first, followed immediately by the target output columns.
  • Testing Data: Your testing CSV must contain exactly Input Nodes columns. No target columns should be included.
  • Data Types: Aside from an optional header row, all data must be numerical. Categorical data (like "Red", "Blue") must be one-hot encoded or numerically mapped prior to upload.

Underlying Theory

Biological Inspiration

Artificial Neural Networks are computing systems loosely inspired by the biological neural networks that constitute animal brains. The app visualizes nodes (artificial neurons) connected by edges (synapses). Each edge transmits a signal scaled by a specific weight, similar to synaptic strength, allowing the network to "learn" complex patterns by continually adjusting these connections.

Architecture Design

The network is structured in layers:

  • Input Layer: Receives the raw numerical data. The number of nodes must match the number of features in your dataset.
  • Hidden Layers: Intermediate processing stages where the network extracts features and patterns. Deep networks (multiple hidden layers) can model highly complex, non-linear relationships.
  • Output Layer: The final prediction. For a binary classification, this might be 1 node (outputting a probability between 0 and 1). For predicting an exact numerical value (regression), this is typically 1 node with a Linear activation.

Mathematics of Forward Propagation

During a forward pass, data moves from left to right. For each node in a subsequent layer, the network calculates the weighted sum of all incoming signals, plus a bias term. Mathematically, for a layer matrix: Z = W · A + b. This result Z is then passed through an Activation Function.

Role of Activation Functions

Without activation functions, a neural network (no matter how many layers it has) would just be a linear regression model. Activation functions introduce non-linearity, allowing the network to learn complex shapes and boundaries.

  • Sigmoid: Squashes values between 0 and 1. Excellent for probabilities but can suffer from vanishing gradients.
  • Tanh: Squashes values between -1 and 1. Often performs better than Sigmoid for hidden layers as data is centered around zero.
  • ReLU (Rectified Linear Unit): Outputs the input directly if positive, otherwise outputs zero. Highly efficient and the modern default for hidden layers.
  • Linear: No alteration to the data. Typically used exclusively on the Output layer for regression tasks.

Training: Cost and Backpropagation

The network learns by comparing its predictions to the actual target answers using a Cost Function (in this app, Mean Squared Error). Once the error is calculated, the network performs Backpropagation—moving backward from output to input, using calculus (the chain rule) to calculate the gradient of the error with respect to every single weight and bias.

Finally, an optimization algorithm (Gradient Descent) updates the weights by taking a small step (determined by the Learning Rate) in the opposite direction of the gradient, slowly minimizing the cost over thousands of Epochs.
